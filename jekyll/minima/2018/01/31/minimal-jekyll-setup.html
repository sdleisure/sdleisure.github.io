    Minima(l) Jekyll Setup

    Jan 31, 2018

        Given this is my first Jekyll site, it seems appropriate to cover its basic setup and deployment to GitHub Pages. GitHub Pages is a hosting service for static site generators. A static website is a good choice if you have no need for client input or if you are a simpleton new to web design, like me. Quick remark: the sequel will have a Debian flavor.

    Installing Git

        First off, we are going to need Git. If you have not already installed it, you can personalize the following:

    $ sudo apt-get -y install git-all
$ git config --global user.name "Foo Bar"
$ git config --global user.email baz@qux.com
$ git config --global core.editor foobar

    The core.editor variable allows you to set your preferred editor, for example, Vim or Emacs. If unset, the system’s default editor will be used. Git can also summarize your configuration.

    $ git config --list

    Installing Jekyll

        Before we can generate our Jekyll site, we must have Ruby installed.

    $ sudo apt-get update
$ sudo apt-get -y install ruby ruby-dev make gcc

    We can now install Jekyll and the Ruby gems dependency manager Bundler.

    $ sudo gem install jekyll bundler

    Creating the Site

        As it turns out, building a Jekyll site with the Minima theme is super hard.

    $ jekyll new mysite && cd mysite
$ git init
$ jekyll build
$ git add . && git commit -m "Initial page commit."

    The next snippet allows you a local glimpse of your site by browsing to http://localhost:4000.

    $ bundler exec jekyll serve --watch >/dev/null 2>&1 &

    The --watch switch performs an auto-regeneration as you locally edit your site.

        It is worth mentioning you can keep your local Jekyll environment in order with that of GitHub Pages. In your site’s Gemfile, just replace the line which contains gem "jekyll" with the following line:

    Run bundler install to finish. Do not forget to keep your locale in sync.

    $ sudo bundler update github-pages

    Deploying the Site

        The time has come to deploy our site to GitHub. Make sure you have created the necessary repositories on GitHub. You can publish a User Page as follows:

    $ git checkout master
$ git remote add origin https://github.com/username/username.github.io
$ git push -u origin master

    Now, your User Page will be hosted at https://username.github.io.

        You can also publish a Project Page with the gh-pages branch.

    $ git branch -m master gh-pages
$ git remote add origin https://github.com/username/repo
$ git push -u origin gh-pages

    You Project Page will appear at https://username.github.io/repo. Separating your site from your project source files can be done with an orphan branch.

    $ git checkout --orphan gh-pages
$ git remote add origin https://github.com/username/repo
$ git push -u origin gh-pages

    At long last, your site should be up for someone everyone to see.
