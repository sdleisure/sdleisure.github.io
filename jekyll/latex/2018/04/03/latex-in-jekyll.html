    Pushing to GitHub

        By way of security policy, GitHub runs only a select few plugins. Thus, we must push our locally generated site to GitHub if we want to utilize the Jekyll-Scholar plugin. For what follows, we assume a develop branch exists in the local Git repository.

    $ git branch develop

    After generating your Jekyll site, the contents of the _site directory must be pushed to GitHub using a site branch.

    $ git checkout develop
$ bundler exec jekyll build
$ git add -A && git commit -m 'Generated site'
$ git checkout -b site
$ git filter-branch --prune-empty --subdirectory-filter _site/ site
$ git remote add origin https://github.com/user/repo
$ git push origin site:master

    When editing the develop branch, the changes must be added to the site branch and pushed.

    $ git checkout develop
$ # Making some changes.
$ bundler exec jekyll build
$ git add -A && git commit -m 'Committing changes'
$ git checkout site
$ git reset --hard refs/heads/develop
$ git filter-branch -f --prune-empty --subdirectory-filter _site/ site
$ git push origin site:master

        Using MathJax, we can begin displaying mathematics in our posts. Regardless of our bias towards \(\rm\LaTeX\), MathJax offers support for MathML and AsciiMath as well.
