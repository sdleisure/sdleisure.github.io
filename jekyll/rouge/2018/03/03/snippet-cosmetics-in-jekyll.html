    Snippet Cosmetics in Jekyll

    Mar 3, 2018

        After tailoring Rouge for my Jekyll site, a memorandum is due. Rouge is a syntax highlighter written in Ruby. As of Jekyll 3, it is the default replacing the Pygments highlighter. It can greatly elevate the display and readability of your code snippets if properly leveraged. What I mean to say is your code can look good even if it stinks.

    Jekyll Configuration

        Rogue is the built-in code highlighter for Jekyll 3. If you have strayed from the default, you will need to make sure Rouge is installed. A compatible Markdown engine is also required.

    $ sudo gem install kramdown rouge

    You can then set Rouge as the highlighter in your _config.yml file.

        Rouge is shipped with the rougify command line utility. The rougify command allows you to set the display theme of your code snippets. First take a look at the available themes.

    $ rougify help style

    To use a particular style, use rougify to create a style sheet and add it to the assets of your site source directory.

    $ rougify style foobar > assets/css/highlight.css

    Then add a link element to the HTML head.

    Code Highlighting

        With Rouge configured, we can start rendering our code blocks. Rouge supports many languages. Enter the following for a list of supported languages:

    $ rougify list

    Code snippets can be created with Markdown.

    #include <iostream>

int main() {
  std::cout << "Hello World!";
}

    Alternatively, Liquid can be used.

    public class HelloWorld {
  public static void main(String[] args) {
    System.out.println("Hello World!");
  }
}

    Liquid can also include line numbers with the linenos argument.
